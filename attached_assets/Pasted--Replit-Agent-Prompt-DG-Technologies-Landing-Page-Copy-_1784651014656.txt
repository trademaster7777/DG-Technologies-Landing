Build a single-page marketing website for **DG Technologies**, a technology services firm that builds affordable websites for small businesses, entrepreneurs, and solopreneurs — with the entire process done over the phone.

## Tech requirements
- React + Vite + Tailwind CSS + **Framer Motion** for all animations, **Lenis** for buttery smooth scrolling
- Fully responsive (mobile-first — most visitors will be small business owners on phones); animations degrade gracefully on mobile and respect `prefers-reduced-motion`
- All CTA buttons link to a booking placeholder: `#book` anchor for now, structured so I can swap in a Calendly/phone link later in one place
- Keep it performant despite the animations: lazy-load below-the-fold sections, GPU-friendly transforms only (translate/opacity/scale), no layout-thrashing scroll listeners
- Footer: dark, spacious, with company name, "© 2026 DG Technologies", placeholder Privacy Policy and Terms links, and a final one-line CTA

## Design direction — PREMIUM IS THE POINT
This site must look like a $10,000+ agency build. It IS the portfolio — visitors should think "if their own site looks like this, imagine what they'll build for me." Reference the feel of linear.app, vercel.com, and stripe.com: dark, cinematic, precise.

- **Aesthetic:** dark-mode-first. Near-black background (#0A0A0F range) with subtle noise/grain texture, electric blue → violet gradient accents, thin 1px borders with low-opacity white (`white/10`) for card edges
- **Hero background:** animated gradient mesh / aurora effect slowly drifting behind the headline (CSS or canvas, subtle, ~60s loop) plus a faint dot-grid pattern that fades toward the edges
- **Typography:** oversized, tight-tracking display headings (Inter or Manrope, weight 700–800, `tracking-tight`), fluid `clamp()` sizing so the H1 feels massive on desktop. Gradient text treatment on key headline words
- **Glassmorphism sticky header:** transparent at top, blurs (`backdrop-blur`) and gains a hairline bottom border once scrolled
- **Cards:** glass effect (translucent bg + blur), soft glow on hover, slight lift (`translateY(-4px)`) with a spotlight/radial highlight that follows the cursor on desktop
- **Buttons:** primary CTA has a gradient fill with animated glow/shine sweep on hover; magnetic hover effect (button subtly follows cursor) on desktop only

## Animations & interactions (make these deliberate, not gimmicky)
- **Page load:** hero headline reveals with a staggered word-by-word rise + fade (Framer Motion staggerChildren); trust bar items cascade in after
- **Scroll reveals:** every section animates in once on scroll (fade + 24px rise, ~0.6s ease-out, staggered children). Use `whileInView` with `once: true`
- **How It Works:** steps connected by a vertical line that "draws" itself as you scroll (SVG path animation tied to scroll progress); each step card slides in alternating from left/right
- **Pricing cards:** rise in with stagger; "MOST POPULAR" badge gets a subtle pulse; The Presence card has a slow animated gradient border
- **Stats/trust moments:** any numbers (e.g., "$999") count up when scrolled into view
- **Parallax:** hero background layers move at slightly different scroll speeds; decorative gradient orbs drift subtly in section backgrounds
- **FAQ accordion:** smooth height animation with rotating chevron
- **Micro-interactions:** nav links get animated underline on hover; scroll progress indicator (thin gradient bar) fixed at the very top of the viewport
- **Restraint rule:** max one "wow" element per viewport. Motion should feel expensive and controlled — 200–600ms, custom cubic-bezier easing, never bouncy or toy-like

## Do NOT
- Add a blog, multi-page routing, or CMS
- Use stock photos anywhere — this design is built on typography, gradients, glass, and motion
- Use cheesy effects: no bouncing elements, no spinning logos, no autoplaying carousels, no custom cursors that hide the real cursor
- Let animations block reading — content must be legible immediately, motion is garnish
- Add popups or chat widgets
- Invent testimonials, phone numbers, or addresses — use the placeholders exactly as specified. Use ONLY the pricing specified above.
